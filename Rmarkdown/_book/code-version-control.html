6 Code Version Control

Version control systems allow for easy access to prior versions of code, branching for development, and easy of collaboration between programmers. Different version control systems exist across several industries, but it is considered best practice to use a git as your version control system for any coding task.

Git is important to use to keep track of code version histories. Code isnâ€™t static and just written once and executed, but is consistently changing. Version control is necessary to keep track of these updates.

6.1 What is Git?

  • Git is a distributed version control system for code
  • Basic workflow
  • Makes changes to code
    • Add code to git
    • Make a commit! (A short comment about WHY you made those changes)
    • Repeat
